# Deploy Project

## Purpose
Deploy the Facebook Clone to free hosting services.

## When to Use
- Ready to go live
- Want to share project with others
- Building portfolio project

## Deployment Stack
| Service | Purpose | Free Tier |
|---------|---------|-----------|
| Vercel | Frontend hosting | ✅ Unlimited |
| Railway | Backend + MySQL | ✅ $5/month credit |
| Render | Alternative backend | ✅ 750 hours/month |

## Steps

### Option A: Railway (Backend + Database)

#### 1. Prepare Backend
```bash
cd server

# Update package.json scripts
{
  "scripts": {
    "start": "node src/index.js",
    "dev": "nodemon src/index.js"
  }
}
```

#### 2. Create Railway Account
- Go to https://railway.app
- Sign up with GitHub

#### 3. Deploy Backend
```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Initialize project
railway init

# Add MySQL database
railway add

# Deploy
railway up
```

#### 4. Set Environment Variables
In Railway dashboard:
```
DATABASE_URL=<auto-generated>
JWT_SECRET=<your-secret>
CLIENT_URL=<your-vercel-url>
PORT=5000
```

#### 5. Run Migrations
```bash
# Connect to Railway
railway run npx prisma migrate deploy
```

---

### Option B: Render (Backend Alternative)

#### 1. Create Render Account
- Go to https://render.com
- Sign up with GitHub

#### 2. Create Web Service
- Click "New Web Service"
- Connect your GitHub repo
- Set root directory to `server`

#### 3. Configure
```
Build Command: npm install
Start Command: npm start
Environment: Node
```

#### 4. Add Environment Variables
```
DATABASE_URL=<your-mysql-url>
JWT_SECRET=<your-secret>
CLIENT_URL=<your-vercel-url>
```

---

### Deploy Frontend to Vercel

#### 1. Prepare Frontend
```bash
cd client

# Update API URL to use environment variable
# In api.js or wherever you make API calls
const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:5000/api';
```

#### 2. Create Vercel Account
- Go to https://vercel.com
- Sign up with GitHub

#### 3. Deploy
```bash
# Install Vercel CLI
npm install -g vercel

# Login
vercel login

# Deploy
vercel
```

#### 4. Set Environment Variables
In Vercel dashboard:
```
VITE_API_URL=https://your-backend-url.com/api
VITE_SOCKET_URL=https://your-backend-url.com
```

---

### Deploy MySQL to PlanetScale (Alternative)

#### 1. Create PlanetScale Account
- Go to https://planetscale.com
- Sign up with GitHub

#### 2. Create Database
- Click "Create database"
- Choose region closest to you

#### 3. Get Connection String
- Go to "Connect" tab
- Copy connection string

#### 4. Update Environment Variables
```
DATABASE_URL="mysql://username:password@host/database?sslaccept=strict"
```

#### 5. Push Schema
```bash
npx prisma db push
```

## Post-Deployment Checklist

- [ ] Frontend loads without errors
- [ ] Can register new user
- [ ] Can login
- [ ] Can create post
- [ ] Can upload images
- [ ] Can send messages
- [ ] Socket.io connects (check Network tab)
- [ ] Images display correctly

## Common Issues

**CORS Error**
```javascript
// In server/index.js
app.use(cors({
  origin: [
    'http://localhost:5173',
    'https://your-vercel-app.vercel.app'
  ],
  credentials: true
}));
```

**Images Not Loading**
- Check if uploads folder is accessible
- Verify static file serving
- Consider using cloud storage for production

**Database Connection Failed**
- Check DATABASE_URL format
- Verify database is running
- Check IP whitelist
